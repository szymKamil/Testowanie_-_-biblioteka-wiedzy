# Różnice między `TemporalField` i `TemporalUnit` w Java

| Aspekt | TemporalField | TemporalUnit |
| --- | --- | --- |
| **Definicja** | Reprezentuje konkretne pole czasowe, takie jak rok, miesiąc, dzień, godzina itp. | Reprezentuje jednostkę miary czasu, taką jak dni, godziny, minuty, sekundy itp. |
| **Przykłady** | `ChronoField.YEAR`, `ChronoField.DAY_OF_MONTH` | `ChronoUnit.DAYS`, `ChronoUnit.HOURS` |
| **Zakres** | Odpowiada za wartość w określonym zakresie (np. rok 2024, dzień miesiąca 15). | Odpowiada za interwał czasu, np. różnica między dwoma datami w dniach. |
| **Zastosowanie** | Używany do odczytu lub zapisu wartości w obiekcie czasu, np. w `LocalDate`. | Używany do manipulacji czasu (dodawanie/odejmowanie), np. `plus`, `minus`. |
| **Metody** | - `getFrom(TemporalAccessor)` – odczytuje wartość pola z obiektu czasu.  - `rangeRefinedBy(TemporalAccessor)` – określa zakres pola dla danej daty/czasu. | - `addTo(Temporal, long)` – dodaje wartość jednostki do obiektu czasu.  - `between(Temporal, Temporal)` – oblicza różnicę między obiektami czasu w danej jednostce. |

W Javie klasy ChronoUnit oraz Period są częścią pakietu java.time, który został wprowadzony w JDK 8 w celu obsługi dat i czasu. Każda z nich służy do różnych celów związanych z mierzeniem czasu i dat.

ChronoUnit

ChronoUnit to enum, który definiuje jednostki czasu, takie jak dni, godziny czy lata. Jest używany do obliczania różnic między datami i czasami oraz do dodawania lub odejmowania określonych jednostek czasu od daty lub czasu.

Główne cechy:
Statyczny enum z predefiniowanymi jednostkami czasu.
Używany głównie z klasami LocalDate, LocalTime, LocalDateTime, itp.
Obsługuje różne operacje arytmetyczne, jak dodawanie i odejmowanie.
Popularne jednostki w ChronoUnit
ChronoUnit.YEARS – lata
ChronoUnit.MONTHS – miesiące
ChronoUnit.DAYS – dni
ChronoUnit.HOURS – godziny
ChronoUnit.MINUTES – minuty
ChronoUnit.SECONDS – sekundy

Period
Opis
Period to klasa reprezentująca okres między dwiema datami w jednostkach: lata, miesiące i dni. Jest szczególnie użyteczna do obliczania różnic w datach kalendarzowych.

Główne cechy:
Reprezentuje ilość czasu w pełnych latach, miesiącach i dniach.
Nie obsługuje godzin, minut ani sekund (w odróżnieniu od Duration).
Może być tworzony za pomocą metod statycznych lub konstruktorów.

```java
import java.time.LocalDate;
import java.time.Period;

public class PeriodExample {
    public static void main(String[] args) {
        LocalDate startDate = LocalDate.of(2023, 1, 1);
        LocalDate endDate = LocalDate.of(2024, 3, 5);

        Period period = Period.between(startDate, endDate);

        System.out.println("Lata: " + period.getYears());
        System.out.println("Miesiące: " + period.getMonths());
        System.out.println("Dni: " + period.getDays());
    }
}
```

| **Zakres jednostek** | Sekundy, minuty, godziny, dni, miesiące, lata | Lata, miesiące, dni |
| **Użycie** | Obliczenia arytmetyczne na datach/czasach | Różnice między datami |
| **Przykłady jednostek** | `ChronoUnit.DAYS`, `ChronoUnit.HOURS` | `Period.ofDays(10)`, `Period.of(1, 2, 3)` |
