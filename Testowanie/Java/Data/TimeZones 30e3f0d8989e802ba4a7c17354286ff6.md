# TimeZones

| **Klasa** | **Opis** |
| --- | --- |
| `ZonedDateTime` | Data i czas z uwzględnieniem strefy czasowej. |
| `ZoneId` | Reprezentuje strefę czasową (np. `Europe/Warsaw`). |
| `ZoneOffset` | Reprezentuje przesunięcie od UTC (np. `+02:00`). |
| `OffsetDateTime` | Data i czas z uwzględnieniem przesunięcia od UTC. |
| `Instant` | Punkt na linii czasu z precyzją do nanosekund. |

⚙️ Metody i zasady pracy ze strefami czasowymi
1. Uzyskiwanie stref czasowych

```java
ZoneId defaultZone = ZoneId.systemDefault();         // Domyślna strefa czasowa systemu
ZoneId warsawZone = ZoneId.of("Europe/Warsaw");      // Konkretnie określona strefa czasowa
```

1. Konwersja daty i czasu na strefę czasową

```java
LocalDateTime localDateTime = LocalDateTime.now();
ZonedDateTime zonedDateTime = localDateTime.atZone(ZoneId.of("Europe/Warsaw"));
System.out.println(zonedDateTime);
```

1. Konwersja między strefami czasowymi

```java
ZonedDateTime dateTimeInWarsaw = ZonedDateTime.now(ZoneId.of("Europe/Warsaw"));
ZonedDateTime dateTimeInNewYork = dateTimeInWarsaw.withZoneSameInstant(ZoneId.of("America/New_York"));
System.out.println(dateTimeInNewYork);
```

1. Konwersja między strefami czasowymi

```java
Instant now = Instant.now();
System.out.println(now); // Czas w UTC
```

Konwersja Instant na ZonedDateTime:

```java
ZonedDateTime zonedNow = now.atZone(ZoneId.of("Europe/Warsaw"));
System.out.println(zonedNow);
```

✅ Zasady dotyczące stref czasowych
1. **Używaj `ZoneId` i `ZonedDateTime`** do operacji wymagających stref czasowych.
2. **Unikaj `LocalDateTime`** do przechowywania dat globalnych – nie zawiera strefy czasowej.
3. **Używaj `Instant`** do przechowywania dat w formacie UTC.
4. **Uwzględniaj zmiany czasu letniego/zimowego** – `ZonedDateTime` automatycznie to obsługuje.
5. **Zawsze podawaj strefę czasową**, gdy wyświetlasz daty dla użytkowników z różnych regionów.
